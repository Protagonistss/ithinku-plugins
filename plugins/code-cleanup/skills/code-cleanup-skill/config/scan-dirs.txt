# 扫描目录配置
# 格式: 目录路径:类别名:关键词策略
# 空行和 # 开头的行会被忽略
#
# ── 字段说明 ──────────────────────────────────────────────
#
# 目录路径
#   相对项目根目录的路径，如 src/page、src/services
#   脚本会递归扫描该目录下所有代码文件
#
# 类别名（用于输出报告中标记候选类型，可自定义）
#   unused_page       - 未引用的页面
#   unused_view       - 未引用的视图
#   unused_component  - 未使用的组件
#   unused_handler    - 未使用的控制器/路由处理
#   unused_service    - 未使用的服务模块
#   unused_util       - 未使用的工具/辅助模块
#   unused_module     - 通用未使用模块（类别名可自由命名）
#
# 关键词策略（逗号分隔，决定用什么关键词在项目中搜索引用）
#   stem   = 文件名（不含扩展名）
#            如 src/page/login/login.js → 关键词 "login"
#   parent = 父目录名，用于路径片段匹配
#            如 src/page/login/login.js → 关键词 "/login/"
#   tag    = kebab-case 标签名（驼峰自动转短横线，适合组件场景）
#            如 MyCourseItem.vue → 关键词 "my-course-item"
#
#   策略组合影响检测精度：
#   - stem,parent  → 同时搜文件名和目录名，适合页面（路径即路由）
#   - stem,tag     → 同时搜文件名和组件标签名，适合 UI 组件
#   - stem         → 只搜文件名，适合后端模块（service/util/handler）
#
# ── 示例配置 ──────────────────────────────────────────────
#
# 前端项目
src/page:unused_page:stem,parent
src/pages:unused_page:stem,parent
src/components:unused_component:stem,tag
src/views:unused_view:stem,parent
# 后端项目
src/controllers:unused_handler:stem
src/services:unused_service:stem
src/utils:unused_util:stem
src/helpers:unused_util:stem
